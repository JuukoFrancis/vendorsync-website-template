App Architecture

System Architecture Diagram

This diagram illustrates the overall system architecture of Vendorsync, showing the main modules and their interactions.

Frontend

The application is built using Flutter for a cross-platform experience across mobile, web, and desktop.

Backend / Database

The backend uses Firebase Firestore for real-time data storage and syncing.

Modules

  • POS
  • Inventory Management
  • Supplier Management
  • Reporting

Auto-Reorder Logic

When stock levels fall below a threshold set by the vendor, the app automatically generates a reorder request to the relevant supplier.

State Management

The app does not use any external state management libraries. It relies on local widget states and setState logic.
